# GRD V4 — Бүрэн шинээр бичсэн хувилбар

## Энэ хувилбарт юу өөрчлөгдсөн бэ

Өмнөх хувилбарууд ChatGPT-ийн бичсэн код дээр суурилж засвар хийгдэж ирсэн. **Энэ V4 бол бүх файл (HTML, CSS, JavaScript, API) эхнээс нь дахин, шинэ зохион байгуулалт, шинэ дизайны системээр бичигдсэн.**

**Database бүтэц өөрчлөгдөөгүй.** `schema.sql`-ийн хүснэгт, багана нэрс өмнөхтэй яг адилхан — учир нь энэ бүтэц аль хэдийн таны D1 database дээр бодитоор бөглөгдсөн байгаа. Хэрэв би үүнийг ч өөрчилбөл таны D1 Console дээр гар аргаар хийсэн бүх ажил дахин хийгдэх шаардлагатай болно. Тиймээс: **schema.sql-г дахин ажиллуулах шаардлагагүй.**

## Функциональ шаардлага (өөрчлөгдөөгүй)

- Ганц нэвтрэх хуудас (`index.html`), role-оор автоматаар чиглүүлнэ:
  - `admin` / `viewer` / `worker` — бүгд эрхийнхээ дагуу зөв хуудсанд очно
- **Dashboard (`dashboard.html`)-ыг нэвтэрсэн хэн ч (захирал, ажилтан хоёулаа) харна.** Гэхдээ тайлан **оруулах** эрх нь өмнөх шигээ хатуу хязгаарлагдсан хэвээр.
- Ажилтан `report.html` дээр зөвхөн өөрийн эрхтэй form-оо харна, эрхгүй зүйл огт харагдахгүй.
- `/api/daily`, `/api/monthly` endpoint бүр нэвтрэлт шаардана (нээлттэй биш).

## Шинэ дизайны систем

ChatGPT-ийн ерөнхий "цайвар SaaS" загвараас татгалзаж, **уурхайн хяналтын самбар (control-room instrument panel)** гэсэн тодорхой концепцоор барьсан:

- **Өнгө:** нүүрсний өнгөт graphite/coal дэвсгэр дээр аюулгүй ажиллагааны amber (`#E3A538`) дохио өнгө — өргөн хэрэглэгддэг "хар дэвсгэр + neon" загвараас ялгаатай, харин уурхайн жилет, анхааруулгын гэрлийн өнгөнөөс санаа авсан.
- **Үсгийн фонт:** гарчиг/тоонд `Big Shoulders Display` (Chicago-гийн ган үйлдвэрийн тэмдэгтэй загварчлагдсан condensed фонт — хүнд үйлдвэрийн сэдэвтэй шууд холбоотой), үндсэн бичвэрт `IBM Plex Sans`, тоон өгөгдөлд `IBM Plex Mono` (tabular figures — жинлүүрийн заалт мэт).
- **Гарын үсэг элемент:** карт/самбар бүрийн дөрвөн буланд хэмжих хэрэгслийн bezel мэт L хэлбэрийн тэмдэг (`.bezel` класс) — "энэ бол шууд заалт" гэдгийг илэрхийлнэ.
- **Модуль код:** PRD / TRN / FUE / EQP / CMP / HSE / ISU — жинхэнэ уурхайн техникийн nameplate-ийн код шиг товчлол, чимэглэл биш, тухайн модулийг таних зориулалттай.
- **Төлөвийн гэрэлтүүлэг:** тайлан ирсэн эсэхийг заах `status-dial` — control panel дээрх индикатор гэрэл (dim/lit) шиг ажиллана.

## File бүтэц

```
index.html          — нэвтрэх (access panel)
dashboard.html       — хяналтын самбар (бүгд харна)
report.html          — ажилтны тайлангийн form
admin.html           — placeholder
style.css            — дизайны систем (CSS variables, bezel, dial, cards)
app.js               — CONFIG / SESSION / API / UI / PAGE.* модулиуд
schema.sql           — D1 бүтэц (аль хэдийн ажилласан тул дахин ажиллуулах шаардлагагүй)
wrangler.toml
functions/api/[[path]].js
```

## Deploy хийх дараалал

1. GitHub repo дээрх бүх файлыг эдгээрээр **бүрэн overwrite** хийнэ.
2. Push хийнгүүт Cloudflare Pages автоматаар deploy эхлүүлнэ.
3. **D1 холболт өөрчлөгдөөгүй** тул дахин тохируулах шаардлагагүй — өмнөх шигээ Settings → Bindings дээр `DB` → `govi_dashboard` бэлэн байгаа бол болоо.
4. `schema.sql`-г **дахин АЖИЛЛУУЛАХГҮЙ** (дээрх анхааруулгыг үзнэ үү).
5. Deploy амжилттай болсны дараа шалгах:
   - `/api/options` → 7 тайлангийн төрөл JSON-оор гарна
   - `admin / 9999` → Dashboard, бүх dial "○ Хүлээгдэж байна"
   - `teever / 2222` → шууд Тээврийн form, бөглөөд илгээнэ
   - `admin`-аар буцаж орох → Тээврийн dial ногоон болж, KPI карт дээр тоо гарна
   - `teever`-ээр дахин орж, "Хяналтын самбар" товч дарах → мөн адил Dashboard-ыг харна
