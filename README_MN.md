# GRD V3 — Ганц нэвтрэлт + Удирдлагын Summary Dashboard

## V2-оос юу өөрчлөгдсөн бэ

**Гол зарчим:** Систем хоёрхон зүйл хийнэ.
1. **Захирал / удирдлага** нэвтрэнгүүт → шууд **Dashboard summary** харна.
2. **Ажилтан** нэвтрэнгүүт → шууд **өөрийн тайлангийн form** харна.

### Өөрчлөлтүүд

- **Ганц нэвтрэх хуудас.** `report.html` дээрх давхар login устгагдсан. Бүгд `index.html`-ээр нэвтэрнэ. Систем role-оор нь автоматаар чиглүүлнэ:
  - `admin` / `viewer` → `dashboard.html`
  - `worker` → `report.html`
- **Dashboard одоо бодит data харуулна.** Нэвтрэнгүүт өнөөдрийн:
  - **Тайлангийн ирц** — 7 хэлтэс тус бүр тайлангаа өгсөн эсэх (✓ хэн, хэдэн цагт / ○ хүлээгдэж байна). Ирсэн тайлан дээр дарвал дэлгэрэнгүй хүснэгт гарна.
  - **Гол үзүүлэлтүүд** — 24 цагийн бүтээгдэхүүн, тээвэрлэсэн нийт тонн, түлшний зарлага/үлдэгдэл, ажилласан/засвартай техник, нийт хүн хүч, ХАБ зөрчил, асуудал. ХАБ зөрчил эсвэл нээлттэй асуудал байвал карт улаанаар анхааруулна.
  - **Сарын нэгтгэл** — сонгосон сарын нийлбэр дүн (бүтээгдэхүүн, тээвэр, түлш, пүү, ХАБ, асуудал).
  - Огноог `‹ ›` товчоор урагш/хойш шилжүүлнэ.
- **Ажилтны хуудас хялбаршсан.** Нэвтрэнгүүт шууд огноо + өөрийн form. Ганцхан эрхтэй ажилтанд сонголтын хэсэг ч харагдахгүй — шууд form гарна.
- **`/api/daily` нээлттэй байсан цоорхой хаагдсан.** Одоо dashboard-ийн бүх data endpoint (`/api/daily`, `/api/monthly`) нэвтрэлт шаардана.
- **Summary-г бүгд харна.** Захирал ч, ажилтан ч нэвтэрсний дараа Dashboard-ыг харж болно. Ажилтан тайлангийн хуудаснаас «Dashboard» товчоор, Dashboard-аас «Тайлан оруулах» товчоор хооронд нь шилжинэ. Гэхдээ тайлан **оруулах** эрх нь хэвээрээ — хүн бүр зөвхөн өөрийн эрхтэй form-оо бөглөнө.
- **Нэг өдөр давхар тайлан орсон тохиолдолд** хамгийн сүүлд шинэчилсэн хувилбарыг dashboard авна.

## Deploy хийх дараалал

1. GitHub repo дээрх файлуудыг эдгээрээр **бүрэн overwrite** хийнэ.
2. Cloudflare Pages deploy success болтол хүлээнэ.
3. D1 binding: `DB` → `govi_dashboard` (өмнө хийсэн бол дахин хийх шаардлагагүй).
4. `schema.sql` өөрчлөгдөөгүй тул **database-ээ дахин ажиллуулах шаардлагагүй** — өмнөх data хэвээр үлдэнэ. (Шинээр эхлүүлэх бол D1 Console дээр `schema.sql`-г ажиллуулна.)
5. `wrangler.toml` дотор `database_id` бөглөгдсөн эсэхээ шалгана.
6. Test:
   - `admin / 9999` → dashboard, өнөөдрийн ирц харагдана
   - `teever / 2222` → шууд тээврийн form гарна, бөглөөд илгээнэ
   - Дахин `admin`-аар ороход тээврийн chip ✓ ногоон болж, картанд тоо гарсан байна

## Дараагийн шатны ажлууд

- Admin хуудас: хэрэглэгч нэмэх / идэвхгүй болгох / PIN солих / эрх тохируулах
- PIN-г browser-т хадгалахын оронд server token (аюулгүй байдал)
- Асуудлын module-ыг тусдаа `issues` хүснэгт болгож олон асуудал, нээх/хаах урсгалтай болгох
- Түлшний орлого/зарлага/үлдэгдлийн тулгалт, сөрөг үлдэгдлийн warning
- Executive Summary-гийн KPI-г захирлын саналаар нарийвчлах
